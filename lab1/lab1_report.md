# 2. Промпт для LLM

## Исходный промпт
Создай Telegram-бота на Python с использованием библиотеки python-telegram-bot.

Функционал бота:
- Бот должен принимать от пользователя задачи в текстовом виде, запрашивать у него приоритет задачи, дату дедлайна, отправлять напоминание о задаче за 14 дней до дедлайна, 7 дней и в день дедлайна.
- Команды, которые должны быть в боте: Поставить задачу, установить приоритет задачи, поставить дату дедлайна, посмотреть все задачи
- Бот должен отвечать дружелюбно, позитивно
- Должен хранить данные: текст задачи, дата дедлайна, приоритет задачи

Требования:
- Бот должен быть простым и понятным
- Код должен быть хорошо прокомментирован
- Использовать файл для хранения данных (JSON или SQLite)
- Добавить обработку ошибок
- код должен работать с первого раза

Создай:

1. Файл bot.py с кодом бота
2. Файл requirements.txt с зависимостями
3. Файл README.md с инструкцией по запуску
4. Файл .env.example для примера конфигурации


## Итерации и улучшения (если были)
1. Изначально в промт был добавлен пункт "- код должен работать с первого раза" т.к. это уже десятая попытка создать бота на третьем аккаунте в курсоре (еще были попытки в gemini, chatgpt, deepseek, AI Google collab)
2. Была добавлена функция "удалить задачу" т.к. при тестировании стало понятно, что ее не хватает для оптимизации хранения данных

## Итоговый промпт
Не менялся кроме дополнительных запросов по исправлению ошибок и добавления доп. функции.

# 5. Трудности и решения
## С какими проблемами столкнулись
Сначала ставила очень длинную задачу с большим функционалом – Курсор справился, но в процессе починки багов бот бесповоротно сломался. Обобщая, проблема одна – нейросети не справляются со слишком амбициозными задачами, а gpt, deepseek, gemini, google collab просто не справляются

## Как их решили
Упрощение функционала и отказ от доведения результата до идеальной картинки из головы (с кнопками, возвратом в меню и т.д.)
